## 关键去重成果

### KEYWORDS 三处重复 → 单一源

| 之前位置 | 形式 | 处理方式 |
|---|---|---|
| admin CardsTab inline | `['mind-clarity', 'heart-resilience', ...]` (id 数组) | 改 `import { ALL_KEYWORD_IDS as KEYWORDS }` |
| apps/api constants.js | `ALL_KEYWORDS = ['Clarity', 'Resilience', ...]` (slug 数组) | 改 re-export `ALL_KEYWORD_SLUGS as ALL_KEYWORDS` |
| apps/api generate-card.js | `KEYWORD_TO_ID = { Clarity: 'mind-clarity', ... }` (slug→id 映射) | 改 `slugToId()` 函数调用 |
| apps/api generate-card.js | `ID_TO_KEYWORD = Object.fromEntries(...)` (id→slug 反向映射) | 改 `idToSlug()` 函数调用 |

四处不同形式的 KEYWORDS 数据，现在统一为 core 的 KeywordModel：

```ts
KEYWORDS: ReadonlyArray<{ id: KeywordId; slug: KeywordSlug; domain: Domain }>
slugToId / idToSlug / domainOf  // 三个 lookup 函数
ALL_KEYWORD_IDS / ALL_KEYWORD_SLUGS  // 两种数组形式
```

### admin inline types 13 处 → core import

8 个 Tab + 5 个子页面的 inline `type X = {...}` 全部替换为 `import type { ... } from '@novame/core/types'`。UI-local 类型（CardForm / StatusFilter / Period / Dashboard 等）保留原位。

### apps/api constants.js 240 行 → 72 行薄壳

所有数据 + 函数搬到 core，原文件保留为 re-export 层，所有现有 import 路径继续工作。

**低效（1.5 启动时避免）**：
- shell heredoc 中的 echo 装饰行（含 `=` / `---`）多次卡住 zsh，**禁用所有装饰行**
- perl `'@novame/...'` 嵌套引号反复出错（`@` 符号 + shell quoting 至少三种翻车），**含 `@` 的字符串改用 view+str_replace 或 sed**
- perl `\{[^}]*\}` 在嵌套大括号上失败（reports.Report / generate-card.KEYWORD_TO_ID），**嵌套结构用 awk 流式处理**
