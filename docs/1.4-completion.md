## 关键去重成果（第一轮的真实价值）

### KEYWORDS 三处重复 → 单一源

| 之前位置 | 形式 | 处理方式 |
|---|---|---|
| admin CardsTab inline | `['mind-clarity', ...]` (id 数组) | 改 `import { ALL_KEYWORD_IDS as KEYWORDS }` |
| apps/api constants.js | `ALL_KEYWORDS = ['Clarity', ...]` (slug 数组) | 改 re-export `ALL_KEYWORD_SLUGS as ALL_KEYWORDS` |
| apps/api generate-card.js | `KEYWORD_TO_ID = { Clarity: 'mind-clarity', ... }` | 改 `slugToId()` 函数调用 |
| apps/api generate-card.js | `ID_TO_KEYWORD = Object.fromEntries(...)` | 改 `idToSlug()` 函数调用 |

四处不同形式现在统一为：

```ts
KEYWORDS: ReadonlyArray<{ id: KeywordId; slug: KeywordSlug; domain: Domain }>
slugToId / idToSlug / domainOf
ALL_KEYWORD_IDS / ALL_KEYWORD_SLUGS
```

### admin inline types 13 处 → core import

8 个 Tab + 5 个子页面的 inline `type X = {...}` 全部替换为 `import type` from core。UI-local 类型（CardForm / StatusFilter / Period / Dashboard 等）保留原位。

### apps/api constants.js 240 行 → 72 行薄壳

所有数据 + 函数搬到 core，原文件保留为 re-export 层，所有现有 import 路径继续工作。

## 协作复盘

**有效**：
- 决策驱动：7 个决策先定再动手，范围从"4 个包"缩到"1 个包"，省 60% 工作量
- 调研先行：每 step 前 grep / view 真实代码，不靠记忆和假设
- 增量验证：每批改动跑 type-check，每个 commit 独立 revertable
- 拒绝顺手重构：发现 OverviewTab Dashboard/Stats 应该是 UI-local（之前被错误标记为 TODO(1.4)），坚决不抽

**低效（下次避免）**：
- shell heredoc 中的 echo 装饰行（含 `=` / `---`）多次卡住 zsh，**禁用所有装饰行**
- perl `'@novame/...'` 嵌套引号反复出错（`@` 符号 + shell quoting 至少三种翻车），**含 `@` 的字符串改用 view+str_replace 或 sed**
- perl `\{[^}]*\}` 在嵌套大括号上失败（reports.Report / generate-card.KEYWORD_TO_ID），**嵌套结构用 awk 流式处理**
- **本报告第一版漏掉了原计划 1.4 包含 5 个项目这件事**，只描述了完成的部分，没明确"未完成"也属于 1.4。下次写完成报告**必须先把原计划清单逐项核对**，再决定标记"完成 / 部分 / 未做"。
